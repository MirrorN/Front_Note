## 样式初始化部分

有些网站css初始化方式：

```
*{
  margin: 0;
  padding: 0;
}
```

有些网站则使用使用更具体的方式：

```
html,body,ul,li,ol,dl,dd,p,h1,h2,h3,h4,h5,h6,form,fieldset,legend,img{
  margin: 0;
  padding 0;
}
```

前者方式写法简单，但是网页渲染速度会比后者慢一些。

注意模块化开发方式：

- 将一个项目按照功能划分，一个功能一个模块，互不影响
- 例如页面的头部和底部，大部分页面都具有相同的部分，将这些部分单独作为一个模块，重复使用
- 创建 common.css 文件，公共样式里面包含 版心宽度，清除浮动，页面文字颜色等公共样式

## Favicon图标

`favicon.ico`图标通常作为缩略的网站图标，显示在浏览器地址栏或者标签上，目前主要的浏览器都支持。

使用流程：

1. 制作favicon图标：

   1. 首先将图标图片切成png图片

   2. 将png图片转化为ico图标，可以使用第三方网站：

      ```
      https://www.bitbug.net/
      ```

2. 将favicon图标放在网站根目录下（直接放在根目录）

3. 在HTML网页中引入favicon图标（在head标签之间）

   ```
   <link rel="shortcut icon" href="favicon.ico" type="image/x-icon"/>
   ```

   

## 网站TDK三大标签与SEO优化



SEO（Search Engine Optimization）搜索引擎优化，是一种利用搜索引擎的规则提高网站在有关搜索引擎内自然排名的方式。其目的在于对网站进行深度优化，从而帮助网站获取免费的流量，在搜索引擎上提升网站排名，提高网站知名度。

使用三个标签来复合SEO优化（TDK）：

- title
- description
- keyword

### 1 title 标题

title具有不可替代性，是第一个重要标签，建议格式：
**网站名（产品名）- 网站介绍 （尽量不要超过30个汉字）** 

例如：

- 京东(JD.com) - 综合网购首选-正品低价、品质保障、配送及时、轻松购物！
- 小米商城 - 小米5S、红米Note4、小米MIX、小米笔记本官方网站

```html
<title>品优购商城-综合网购首选-正品低价、品质保障、配送及时、轻松购物！</title>
```



### 2 description 网页介绍

简要说明网站时做什么的，建议将 description 作为网站总体业务和主题概括，多采用”我们是“、”我们提供“、”xxx网作为“、”电话：“之类语句

例如：

```html
<meta name="description" content="品优购商城-专业的综合网上购物商城，销售家电、数码通讯、电脑、家居百货、服装服饰、母婴、图书、食品等数万个品牌的优质商品。边界、诚信的服务，为您提供愉悦的网上购物体验！">
```



### 3 keywords 关键字

页面关键字时搜索引擎的关注点之一。

keywords最好**限制在 6-8 个，关键词之间使用英文逗号隔开**，采用关键词1,关键词2的形式。

```html
<meta name="Keywords" content="网上购物,网上商城,手机,笔记本,电脑,MP3,CS,VCD,相机,数码,配件,手表">
```



## 关键内容



### 1. LOGO 的 SEO 优化

专业写法：

1. logo里面首先放一个 h1 标签，目的是为了提高权重，告诉搜索引擎这个地方很重要
2. h1 标签里面放一个链接（链接回到首页），将logo的背景图片给链接
3. 为了搜索引擎收录，在链接里面放文字（网站名称），但是文字不要显示
   1. text-indent 移到盒子外边（text-indent: -9999px），然后 overflow：hidden（淘宝做法）
   2. font-size 设置为0（京东做法）
4. 最后给链接一个title属性，鼠标放在logo图上的时候可以看到提示文字



### 2.搜索框部分

注意点：

1. 设置高度的时候要使用精确值，不要设置为 100%
2. 对于 input 标签和 button 标签，由于是行内块元素，所以直接使用会出现空隙，使用 浮动属性 或者 定位 去除
3. 注意去掉边框的大小



### 3. 购物车模块

- 类似电商网站显示的小红圈标志，这部分在显示上会向父盒子上偏移，所以可以使用绝对定位做
- 这部分不要设置宽度，实际中可能数字较大，所以只设置高度，让实际的文本撑开盒子
- 对于绝对定位的位置，这里使用 left 属性设置，这样如果数字较大，内容会自左向右撑开，反之如果设置的是right属性，则会向左撑开
- 如果存在文本的位置不对 -- 考虑是否因为继承改变了 line-height 等属性



### 4. 导航栏模块

- 对于 `hover`设置鼠标移动的效果改变效果，要针对具体的元素进行设置：

  ```css
  .dropdown .dd ul li:hover a {
      color: #b1191a;
  }
  ```

  上述效果表示鼠标指针移动到 li 元素的时候， 对于 a 元素的样式改变

- 对于列表形式的链接，经常使用 li 元素与 a 元素搭配使用的方式，那么在一些样式的设置上，如果**希望用户在一定范围内都可以点击链接，最好对  a 元素的 padding 元素进行设置**



### 5. 底部模块

- 大部分内容都是使用列表进行排列的

- 注意此处的小 logo 可以使用精灵图技术配合 background 元素进行 设置：

  ```css
  .mod_service .icon1 {
      background: url(../images/icons.png) no-repeat -252px -2px;
  }
  ```

- 对于多个最后多个列表形式的内容，使用 dl 元素搭配浮动进行设置

- 对于列表中特殊位置的元素，注意使用 `:nth-child`方法：

  ```css
  .mod_help dl:last-child {
      width: 90px;
      text-align: center;
  }
  ```

- 对于二维码这样的图片，直接在 dd 元素中插入 img 即可，并且这样直接对父元素设置 `text-align`属性还可以保证所有元素居中对齐



### 6. main主体模块制作

- 对于轮播图部分，由于这部分图片需要经常更换，所以要将该图片放在 upload 文件夹（放产品图片）
- 注意使用标签一定要注意其语义对应，例如某些加粗的条目，可以看作是标题进行着重强调，所以使用 `h`系列标签合适
- 万万注意样式名称不要写错了，否则会有很多意想不到的错误
- 注意右侧网格内容的显示，每个方格只有右边框和下边框（每个方格的大小为63）这样一行4个格子之后会超出父盒子3px
- 书写样式的时候注意样式的优先级，以及书写规范



问题：对于

```html
<div class="div1">
    <ul>
        <li>
            <p>test1</p>
        </li>
        <li>
            <p class="i1">test2</p>
        </li>
        <li>
            <p>test3</p>
        </li>
    </ul>
</div>
```

如果使用样式：

```css
<style>
.div1 li {
    color: red;
}

.div1 li .i1 {
    color: blue;
}
</style>
```

此时样式有效。

而另一种写法：

```css
<style>
.div1 li p {
    color: red;
}

.div1 li p .i1 {
    color: blue;
}
</style>
```

此时样式是无效的，但是根据样式优先级计算后的权重，第二种写法中后者样式权重是更高的，非常奇怪。

### 7. 推荐模块

- 对于明显有规律的部分，使用列表元素制作
- 注意背景图片（images）和商品图片（upload）放置的位置
- 对于图像之间的竖线，根据具体的样式可以使用不同的实现方法：
  - 使用宽度为1px的盒子制作，设置背景颜色，并通过定位设置其位置
  - 直接设置所有的 li 元素的边框，并通过 last-child 去掉最后元素的样式
  - 注意结合 nth-child(-n+k) 这类方法以及伪元素 
- 对于列表项，如果**直接对 li 元素设置右浮动，会出现列表项自右向左排列的问题**，解决方法：
  - 首先对父元素设置右浮动
  - 然后对 li 元素设置左浮动
- 注意标签语义，例如 ‘家用电器“这样的文本一般都是作为标题，因此要使用 h 系列的标签
- 注意很多图片都是作为链接使用的，所以要包含在 a 标签中
- 注意**如果 a 标签中包含有宽度的元素（例如 img），要转换为块级元素才安全**



### 8. tab栏布局需求

需要使用两个盒子完成，上方是放置选项的 `tab_list`，下方是防止内容的 `tab_content`，上下内容一一对应，点击tab_list的选项一，显示tab_content的内容一

### 9. 列表页

注意点：

- 对于商品列表，每张图片经常出现的效果是鼠标移动到上面的时候会出现红色边框，如果直接设置`:hover`为边框设置宽度，会使得图像向下移动，所以正确做法是，首先设置每个 li 的边框为`border: 1px solid transparent;`

- 如果每个li的边框有部分（右侧和下面）被遮挡住，有可能是图片的大小和li的大小一样，因为 box-sizing的设置，使得li的边框会被压住

- 使用`line-heihgt`调整文字的位置的时候（例如div的高度为40，为了让文字向下偏移，将文字的 line-heihgt 设置为43px），这时候下面的div元素会受到多出来的3px的影响会出现位置变化，所以要及时设置 `overflow:hidden`，将多出的部分裁掉

- 注意复杂元素的选择方式：

  ```css
  .sk_con ul li:last-child a::after {
      content: '\e91e';
      font-family: 'icomoon';
      margin-left: 3px;
  }
  ```

  上述代码表示li的最后一个元素中的a元素设置after伪元素内容



### 10. 注册页面

- 注册页面比较隐私，所以为了保护用户的信息，不需要对当前页面做SEO优化，尽量写的简单
- 对于注册表单，首先设置一个父盒子将所有表单包含
- 使用 ul + li 设计表单
- 注意label标签是行内元素，为了整齐显示，应首先转化为行内块元素，然后设置其宽度，最后设置右对齐或者左对齐
- 注意所有的表单元素最后要使用 form 标签包裹，方便提交
- 对于小图标+文本的组合 要注意设置 行内块元素 + `vertical-align:middel`的方法来保证文字与图标垂直方向对齐
- 对于表单验证的提示（例如表示正确错误的小图标等）都要设置单独的类名，方便后续使用 JS 代码控制
